Section 1: Introduction

This is a test page designed to perfect mobile scrolling for our bridge app. We're testing on various devices, especially the Pixel 9a which has been problematic.

The key is getting the modal body to scroll properly while keeping the header and footer fixed. This should work on all mobile devices.

Section 2: Features Grid

Section 3: Technical Details

We use flexbox layout with the modal body set to flex: 1 and overflow-y: auto. The critical CSS property is min-height: 0 which allows the flex item to shrink below its content size.

For iOS devices, we add -webkit-overflow-scrolling: touch to enable momentum scrolling. The scrollbar is styled to be more visible on mobile devices.

Section 4: Browser Compatibility

This solution works across all modern browsers including:

  • Chrome on Android (including Pixel devices)
  • Safari on iOS (iPhone and iPad)
  • Firefox Mobile
  • Samsung Internet
  • Desktop browsers (Chrome, Firefox, Safari, Edge)

Section 5: Implementation Notes

When implementing this in the bridge app, remember to:

  1. Set the modal overlay to fixed positioning
  2. Use flexbox for the modal content container
  3. Make the scrollable area use flex: 1
  4. Add touch-action: manipulation to buttons
  5. Include the webkit scrolling properties
